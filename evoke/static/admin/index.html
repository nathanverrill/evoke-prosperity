Brightspace Mission Sync

This admin login never needs a Brightspace account. Pulling real data does need a live Brightspace connection though — "Connect Brightspace" logs in once, as whichever Brightspace account you connect with, and keeps working after that without logging in again. That account needs to be an instructor/TA in the course for roster and grade pulls to work (assignment pulls are fine even at student level).

An Evoke mission is the container for exactly one Brightspace assignment — create the mission first (below), then link it to one row here. Nothing is written by pulling; linking only attaches an ID to a mission you already created.

Brightspace Roster

Real course enrollment, cross-referenced against who's actually logged into EVOKE yet — visibility only, not an import step (that already happens automatically the first time a student logs in via Brightspace).

Dev Users

Two seeded local accounts standing in for a Brightspace-connected team (Player One + Player Two, same Demo Team) plus this admin login -- enough to test team evidence, reflections, and progress end-to-end without a real tenant. Log in as either from this browser at /?login=player1@evoke.local or /?login=player2@evoke.local. Reset clears that account's progress (evidence, reflections, XP, badges, chat, checkins) back to fresh; team assignment stays put.
